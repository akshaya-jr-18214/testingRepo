Features

Everything you need to validate branch integrity across GitLab repositories

🔍

Two-Phase Extraction

Quick scan of ~500 commits first. If no common changeset is found, automatically falls back to a full deep scan.

✅

Verified Common Changeset

Doesn't just find the first match — verifies the next 4 entries below also match, eliminating false positives.

📊

Detailed Results

Rich results page with statistics, missing IDs, fallback changesets, verification table, and full commit details.

🍒

Cherry-Pick Integration

Select missing commits and trigger cherry-pick directly via Zoho CRM API with one click.

🔔

Owner Notifications

Notify commit owners about missing entries directly from the results page via Zoho CRM functions.

📜

Validation History

Stores the last 100 validations locally in the browser with quick-access from the extension popup.

Open the Extension

Click the extension icon in Chrome toolbar. Select your service, enter the parent and child branch names.

Quick Scan

The extension opens GitLab tabs, scrolls through commits, and extracts ~500 entries per branch to find a common changeset quickly.

Deep Scan (if needed)

If no common changeset is found in the quick scan, a full extraction runs automatically — loading the complete commit history.

Review Results

A detailed results page shows missing task IDs, verification data, statistics, and one-click cherry-pick or notification actions.

Installation

Get started in under a minute

Download

Download the latest release .zip from the GitHub Releases page — or clone the repo.

Unzip

Extract the archive to a folder on your machine. The folder should contain manifest.json at the root.

Load in Chrome

Go to chrome://extensions, enable Developer mode, click Load unpacked, and select the folder.

Validate

Navigate to any GitLab page, click the extension icon, fill in branches, and hit Validate.
